## Part 1 ‚Äî Rust Backend

### Critical

**[RUST2-C1] `import_data` deletes inside transaction then inserts outside ‚Äî atomicity gap ‚Äî commands.rs:848‚Äì899**
The DELETE phase runs inside a SQLx transaction and commits. All INSERTs (holdings, alerts, config, transactions, dividends) run on the bare pool after the commit, outside any transaction. A failure on insert row N leaves the DB empty-then-partially-filled with no rollback possible.

**[RUST2-C2] Dynamic `$N` placeholder in `check_and_trigger_alerts` UPDATE ‚Äî db.rs:881‚Äì895**
The batch UPDATE is assembled by formatting `$1`, `$2`, ... placeholders in a loop. SQLite with SQLx runtime queries expects `?` placeholders, not `$N` positional binding. Depending on SQLx version this silently binds wrong values or panics at runtime.

**[RUST2-C3] `restore_database` overwrites DB file with live pool open and WAL active ‚Äî commands.rs:974‚Äì1053**
`std::fs::copy` replaces `portfolio.db` while the pool holds open file descriptors. The WAL file (`portfolio.db-wal`) and shared-memory file (`portfolio.db-shm`) remain live, and the WAL can be replayed on next open ‚Äî potentially overwriting the restored data with pre-restore writes. Pool is not closed before copy.

**[RUST2-C4] `upsert_symbol_fundamentals` can violate NOT NULL on `name` column ‚Äî db.rs:573‚Äì606**
If the symbol doesn't already exist in `symbol_cache`, the INSERT sets `name`, `asset_type`, `exchange`, `currency` to `NULL`, but the column is `NOT NULL` in the schema. The constraint violation is silently suppressed by `let _ = ...`. The symbol may be left in an inconsistent half-written state.

### High

**[RUST2-H1] Stale price/FX cache returns empty ‚Äî portfolio shows $0 prices ‚Äî db.rs:416‚Äì476**
`get_cached_prices()` and `get_fx_rates()` filter `WHERE updated_at >= datetime('now', '-60 minutes')`. If prices are > 60 minutes old (market closed, no refresh), `build_portfolio_snapshot` uses `cost_basis` as the price, displaying inflated/wrong gain/loss with no user-visible explanation.

**[RUST2-H2] `search_symbols` always hits Yahoo Finance API even when DB cache is fresh ‚Äî commands.rs:671‚Äì696**
The in-memory cache is checked first. If the query missed the in-memory TTL but DB cache has a recent result (e.g., 6 minutes old), the code still calls Yahoo unconditionally rather than using the DB result.

**[RUST2-H3] `fetch_all_fx_rates` has unbounded parallelism ‚Äî fx.rs:29‚Äì46**
`futures::future::join_all` fires one HTTP request per unique currency simultaneously. The analytics path was fixed with `buffer_unordered(5)`, but `fetch_all_fx_rates` (used in `refresh_prices`) is still unbounded.

**[RUST2-H4] `fetch_all_prices` issues unbounded concurrent HTTP requests ‚Äî price.rs:103‚Äì111**
Same pattern as RUST2-H3. `futures::future::join_all(futures)` over all symbols with no concurrency cap. 50 symbols ‚Üí 50 simultaneous Yahoo Finance connections.

**[RUST2-H5] `add_dividend` fetches all holdings to resolve one symbol ‚Äî commands.rs:770‚Äì777**
`db::get_all_holdings(pool)` called to find `holding_id`'s symbol. Should be `SELECT symbol FROM holdings WHERE id = $1`.

**[RUST2-H6] `update_account` fetches all accounts to get `created_at` ‚Äî commands.rs:2118‚Äì2128**
`db::get_accounts(pool)` + linear scan for a single field. Should be `SELECT created_at FROM accounts WHERE id = $1`.

**[RUST2-H7] `get_annual_dividend_income` uses stale/empty FX cache ‚Äî commands.rs:156‚Äì158**
Passes `cached_fx` from `db::get_fx_rates()` (60-min TTL). On first launch or after cache expiry, all multi-currency dividends are converted at 1:1, silently over/understating income.

### Medium

**[RUST2-M1] `Utc::now()` computed per-holding in snapshot loop ‚Äî portfolio.rs:90**
Date can change mid-loop near midnight, causing inconsistent daily P&L inclusion across holdings in the same snapshot.

**[RUST2-M2] `set_config_cmd` accepts arbitrary keys ‚Äî no schema ‚Äî commands.rs:44‚Äì56**
Any string key silently persists. Typos produce permanent orphaned config entries.

**[RUST2-M3] Manual URL-encoding incorrect for multi-byte Unicode ‚Äî search.rs:9‚Äì20**
`format!("%{:02X}", c as u32)` for `√©` (U+00E9) emits `%E9` (Latin-1). Yahoo Finance expects UTF-8 percent-encoding `%C3%A9`. Japanese company name searches return 0 results.

**[RUST2-M4] `get_dividends` INNER JOIN silently drops dividends for externally-deleted holdings ‚Äî db.rs:1075‚Äì1100**
If a holding is deleted externally (FK cascade didn't fire), its dividends vanish from `get_dividends` results without error.

**[RUST2-M5] `backup_database` path-traversal guard doesn't canonicalize final `dest` ‚Äî commands.rs:939‚Äì959**
Only the parent is canonicalized. A path whose final component resolves outside `app_data_dir` (after symlink resolution) bypasses the guard.

**[RUST2-M6] `HoldingWithPrice` duplicates all 18 `Holding` fields ‚Äî types.rs:246‚Äì277**
If `Holding` gains a new field, `HoldingWithPrice` must be manually updated. The recent dividend fields required sync changes across both structs.

**[RUST2-M7] `Dividend.id` is `i64` while all other entity IDs are UUID strings ‚Äî types.rs:392‚Äì410**
Breaks the generic ID handling contract in the frontend. Frontend must handle two ID type patterns.

**[RUST2-M8] `tracing-appender` listed in Cargo.toml but never used ‚Äî Cargo.toml:29**
Adds compile time and binary size for zero benefit. Structured logging only writes to stderr.

**[RUST2-M9] Snapshot range query uses TEXT comparison on timestamps ‚Äî db.rs:684‚Äì708**
`recorded_at >= $1 AND recorded_at <= $2` is lexicographic. Timestamps with non-`+00:00` offsets sort incorrectly.

**[RUST2-M10] `weighted_beta` normalized by holdings-with-beta weight, not full portfolio ‚Äî commands.rs:1451‚Äì1455**
If only 60% of portfolio has beta data, reported beta is for that 60% only, not the full portfolio. Undocumented behavior.

### Low

- **[RUST2-L1]** `make_holding` test helper duplicated in commands.rs, portfolio.rs, csv.rs ‚Äî should be in a shared test module
- **[RUST2-L2]** Acknowledged duplicate tests in `commands.rs:1543‚Äì1545` marked TODO, never removed
- **[RUST2-L3]** `fetch_asset_profile` returns redundant `symbol: String` clone in tuple ‚Äî commands.rs:1132
- **[RUST2-L4]** `get_symbol_metadata_with_cache` and `fetch_asset_profile` are domain logic in commands.rs ‚Äî should live in a `fundamentals.rs` module
- **[RUST2-L5]** SQLx `chrono` feature enabled but all timestamps stored as `TEXT`; `uuid` `serde` feature enabled but UUIDs always converted to `String` ‚Äî both features add build time for no benefit

### Build & Dependencies

- **[RUST2-B2]** SQLx `chrono` feature unnecessary (timestamps stored as TEXT, not mapped via SQLx)

## Part 2 ‚Äî TypeScript Frontend

### Critical

**[TS2-C1] Flash of Unstyled Content (FOUC) on theme load ‚Äî hooks/useTheme.ts:20‚Äì48**
`useTheme()` reads the persisted preference asynchronously inside a `useEffect`. The dark theme defaults apply immediately on mount; users with the light theme preference see a dark‚Üílight flash on every launch. Fix: apply theme synchronously from an inline `<script>` in `index.html` before React renders.

**[TS2-C2] i18n hard-coded to English; language flash on mount ‚Äî lib/i18n.ts:13, hooks/useLanguage.ts:15‚Äì36**
`i18n.ts` initializes with `lng: 'en'` unconditionally. `useLanguage` stores state as `'en'` then asynchronously reads the persisted preference and calls `i18next.changeLanguage()`. Components render once in English before the correct language is applied.

**[TS2-C3] `buildMockSnapshot` totalValue wrong ‚Äî weight overflows ‚Äî hooks/usePortfolio.ts:101‚Äì126**
`totalValue = holdingsList.length * 1000`. A single BTC holding (qty 0.85, costBasis 42000) produces `weight = 35700 / 1000 = 35.7` (3570%). Consumers expecting a 0‚Äì1 fraction receive wildly incorrect values.

**[TS2-C4] `Dividends.tsx` forward-income total sums across mixed currencies ‚Äî components/Dividends.tsx:530‚Äì536**
`estimatedAnnualIncome` is summed across all holdings regardless of `indicatedAnnualDividendCurrency`. The total is displayed in whatever currency the first row has, incorrectly adding CAD + USD + EUR dividend income as a single number.

### High

**[TS2-H1] System preference theme listener re-subscribes every theme change ‚Äî hooks/useTheme.ts:51‚Äì58**
The `matchMedia` listener captures `theme` via closure dep. Registered/removed on every `theme` state change. A brief window exists between removal and re-subscription where OS dark-mode toggles are missed.

**[TS2-H2 through TS2-H7] Mass untranslated components ‚Äî 6 major pages**
All 7 locale files define translation keys for Analytics, Dividends, Dashboard, Alerts, Settings (Appearance + DataManagement sections), but none of these components call `t()`. All strings are hardcoded English:
- `Analytics.tsx` ‚Äî entirely hardcoded ("Portfolio Analytics", "Risk Metrics", "Portfolio Beta", "Sector Breakdown", etc.)
- `Dividends.tsx` ‚Äî largely hardcoded ("Record Dividend", "HOLDING", "AMOUNT / UNIT", etc.)
- `Dashboard.tsx` ‚Äî all stat labels hardcoded ("Cost Basis", "Total Gain/Loss", "Holdings", etc.)
- `Alerts.tsx` ‚Äî section headers and table headers hardcoded
- `Settings.tsx:599‚Äì613` ‚Äî Appearance section hardcoded
- `Settings.tsx:191‚Äì467` ‚Äî DataManagementSection entirely hardcoded
- `TopBar.tsx:19‚Äì31` ‚Äî `ROUTE_TITLES` never passes through `t()`

**[TS2-H8] `weight` vs `targetWeight` unit mismatch in `useActionInsights` ‚Äî hooks/useActionInsights.ts:30‚Äì31**
`holding.weight` is a fraction (0‚Äì1 from Rust) but `targetWeight` is a percentage (0‚Äì100 stored in DB). Drift is `|0.12 - 12| = 11.88` ‚Äî always triggers critical severity for every holding with a target. The mock data stores both as percentages, masking the bug in browser mode.

**[TS2-H9] Date inputs hardcode `colorScheme: 'dark'` ‚Äî AddHoldingModal.tsx:642, Dividends.tsx:100**
Native date picker chrome stays dark when user selects light theme.

**[TS2-H10] TopBar daily P&L is full-portfolio; Dashboard can be filtered by account**
`portfolio.dailyPnl` reflects the entire portfolio even when Dashboard's account filter is active. Mismatch between the visible number in TopBar and the filtered totals in Dashboard panels.

## Part 3 ‚Äî Security, Architecture, CI

### Security

**[ARCH2-S1] CSP explicitly disabled in tauri.conf.json ‚Äî High**
`"csp": null` removes the primary webview-level XSS defence. Combined with `tauriInvoke` being callable from any injected script, XSS ‚Üí `import_data` wipes and replaces the entire database.

**[ARCH2-S2] `add_alert` accepts NaN/Infinity/negative threshold ‚Äî no validation ‚Äî Medium**
`PriceAlertInput.threshold` is passed directly to the DB. `f64::NAN` comparisons always return false (alert never fires); `f64::INFINITY` never triggers; negative thresholds produce nonsensical results.

**[ARCH2-S3] `add_holding`/`update_holding` don't validate `quantity`, `cost_basis`, `currency` ‚Äî Medium**
NaN or infinite values silently persist and corrupt downstream portfolio calculations.

**[ARCH2-S4] `set_config_cmd` accepts arbitrary keys with no allowlist ‚Äî Medium**
An XSS-exploited or buggy frontend can write arbitrary config keys that persist permanently.

**[ARCH2-S5] `restore_database` accepts source paths from any filesystem location ‚Äî Medium**
`source_path` is not restricted to `app_data_dir` or user-chosen safe directories. Combined with null CSP, XSS could read any SQLite-formatted file from disk.

**[ARCH2-S6] 4 high-severity npm vulnerabilities in vitest 4.1.x dev chain ‚Äî High**
`flatted <=3.4.1` in `@vitest/ui` ‚Üí `vitest >=4.1.0-beta.1` ‚Üí `@vitest/coverage-v8`. Dev dependencies only (not in Tauri binary) but CI runs under the vulnerable version.

**[ARCH2-S7] `add_dividend` silently proceeds with empty symbol when holding not found ‚Äî Medium**
Should return user-friendly error before hitting DB FK constraint.

### System Design

**[ARCH2-D1] i18n language hard-coded to `'en'` at startup ‚Äî user preference never loaded ‚Äî Medium**
`i18n.ts:13` sets `lng: 'en'` unconditionally. User preference stored via `set_config_cmd('ui_language')` is never read at startup. Language switch only works within the session if the Settings UI calls `i18next.changeLanguage()`, but is not persisted across restarts.

**[ARCH2-D2] WAL auto-checkpoint never explicitly scheduled ‚Äî Medium**
SQLite's default auto-checkpoint at 1000 pages may allow the WAL file to grow significantly during heavy price-refresh sessions. `backup_database` correctly flushes WAL first (good), but routine operation has no periodic checkpoint.

**[ARCH2-D3] `fetch_all_prices` unbounded concurrent HTTP ‚Äî same issue as analytics (now fixed) ‚Äî Medium**
`price.rs:103‚Äì111` still uses `futures::future::join_all` with no concurrency cap.

**[ARCH2-D4] Migration 0005 disables FK enforcement without per-connection restore guard ‚Äî Low**
`PRAGMA foreign_keys = OFF` in a migration is per-connection; if the migration fails mid-way, FK enforcement stays off for that connection's lifetime. Low risk given SQLx's transaction wrapper.

**[ARCH2-D5] `get_symbol_metadata_with_cache` is domain logic in commands.rs ‚Äî Low**
After the refactor to portfolio.rs/csv.rs, this function and `fetch_asset_profile` remain in commands.rs. Should move to `fundamentals.rs`.

### CI/CD

**[ARCH2-CI1] `actions/checkout@v6` and `setup-node@v6` don't exist ‚Äî all CI runs fail ‚Äî Critical**
`ci.yml` lines 41, 71, 148 reference `actions/checkout@v6` and `actions/setup-node@v6`. Latest stable is `@v4`. Every CI run fails at the checkout step.

**[ARCH2-CI2] `ci-status` roll-up treats `skipped` as pass ‚Äî High**
Path-filter logic skips frontend/backend jobs based on changed files. `ci-status` checks for `failure` only. A PR touching only `src-tauri/**` skips all frontend tests and still gets a green check, allowing TypeScript type errors to merge.

**[ARCH2-CI3] No coverage threshold enforced ‚Äî Medium**
`vitest.config.ts` has no `thresholds` block. CI generates coverage reports but does not fail if coverage drops below the stated 80% target.

**[ARCH2-CI4] Rust CI doesn't use `--locked` ‚Äî Low**
`cargo build` and `cargo test` in CI don't pass `--locked`. Cargo may silently upgrade dependencies beyond what's in Cargo.lock.

**[ARCH2-CI5] E2E suite skipped on backend-only changes ‚Äî Medium**
E2E job has `if: needs.changes.outputs.frontend == 'true'`. A Rust command that changes its response shape won't trigger E2E validation.

### Operational

- **[ARCH2-O1]** Persistent log file not configured ‚Äî `tracing-appender` is listed but unused; logging writes to stderr only (invisible in production)
- **[ARCH2-O2]** App version not exposed to frontend ‚Äî `Settings.tsx` shows hardcoded `"0.1.0"` instead of reading from Tauri runtime
- **[ARCH2-O3]** No migration rollback strategy documented
